## 搬瓦工换机房是什么意思

搬瓦工（BandwagonHost）的换机房功能，官方叫 **"Migrate to another DC"**，入口在 KiwiVM 控制面板里。本质上就是把你的 VPS 整个迁移到另一个数据中心——系统、数据、配置全部保留，换的只是物理位置和 IP 地址。

不收钱，不限次数，但会消耗流量。这三点是这个功能的核心特征，后面会详细说。

## 搬瓦工换机房详细步骤

**第 1 步：登录 KiwiVM 控制面板**

打开搬瓦工官网（国内访问推荐用镜像站），点击顶部 Services → My Services，找到对应的 VPS，点击右侧 "KiwiVM Control Panel" 按钮进入面板。

**第 2 步：找到换机房入口**

面板左侧菜单里找 "Migrate to another DC"，点进去。页面会显示你当前所在的机房，以及所有可以切换的目标机房列表。

**第 3 步：选择目标机房**

从列表里选你想去的机房，点 "Confirm on next step"。

这里有一段官方说明值得重点看：

- 迁移过程中所有数据都会保留，不会丢失
- IP 地址会变更——换了机房就是换了 IP，这是必然的
- 如果你的 VPS 上跑着绑定固定 IP 的服务，迁移完要记得更新配置
- 迁移消耗计入当月流量使用量

**第 4 步：确认开始迁移**

点击确认按钮后，系统开始迁移。整个过程通常在 15 分钟内完成，也可能视 VPS 数据量长达几小时。迁移期间 VPS 基本可用，只有最后阶段会短暂停机几秒到几分钟。迁移完成后会收到邮件通知。

**第 5 步：更新相关配置**

迁移完成后，记几件事：
- 如有绑定域名，更新 DNS 记录到新 IP
- 如有服务配置里写死了旧 IP，逐一更新
- 检查新机房连接是否达到预期

## 流量消耗这件事，很多人忽略了

换机房免费、无限次——但不是真的"无代价"。

每次迁移都会消耗流量，消耗量和 VPS 上已用硬盘大小成正比，大约是已用空间的 4-8 倍。举个例子，VPS 里装了 5GB 数据，迁移一次大概消耗 20-40GB 流量。

这对流量大的套餐影响不太显著，但对月流量只有几百 GB 的入门套餐，反复折腾换机房可能很快把流量用完。

所以实际操作上的建议是：买完先把几个你感兴趣的机房各试一周，找到效果最好的那个后就固定下来，不要没事老换。

## IP 地址在换机房时怎么变

换了机房，IP 必然变。这是设计如此，没有例外。

有一个细节：如果你从机房 A 换到机房 B，再从机房 B 换回机房 A，大概率会拿回原来的那个 IP——前提是在这段时间里没有其他人把那个 IP 用走。

换机房获得新 IP，是处理"IP 被污染"问题的一个思路。但有一个限制：**如果你的 IP 已经被墙（在国内被封锁），换机房功能会被系统自动禁用**。也就是说，IP 被封之后，你没办法通过换机房来绕过去换一个新的 IP。这个时候唯一的官方出路是付费购买新 IP（$19 一次）。

所以如果你发现换机房的选项灰掉了、或者一直提示等待 10-15 分钟，第一件事是去检查当前 IP 是否已经被墙，别一直以为是网络抖动。

## 换机房提示报错怎么办

常见报错有几种：

**"Region is full. Please try again in a few hours. (734104)"**  
目标机房容量已满，临时无法接受迁入。等几个小时再试一般可以解决，也可以换一个别的机房试试。

**"Failed to migrate (734205)"**  
通常是目标机房对某些套餐有额外限制，或者该机房暂时关闭了迁入通道。检查你的套餐是否支持目标机房，或稍后重试。

**"Migration backend is currently not available for this VPS. (734152)"**  
后台正在处理其他任务，稍等一段时间再发起请求即可。

遇到报错先别慌，大部分情况等一两个小时再试都能解决。真解决不了可以提工单，搬瓦工客服是英文工单制，但响应还算及时，碰过复杂问题也能处理。
